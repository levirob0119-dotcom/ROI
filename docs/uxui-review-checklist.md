# UXUI 中期评审清单（组件库改造期）

## 使用方式

- 每完成一个阶段（高频组件、页面块迁移、核心工作台）执行一次评审。
- 执行命令：
  - 启动前端：`npm run dev`
  - 生成截图：`npm run uxui:review`
- 在 `docs/uxui-reviews/<timestamp>/review-summary.md` 记录本轮结果。

## 评审维度（每项 1-5 分）

1. 视觉一致性
- 间距、边框、阴影、圆角是否统一。
- 是否存在旧风格遗留块（突兀的 ring/阴影/渐变）。

2. 表单体验
- 输入、下拉、勾选、开关、单选的状态反馈是否清晰。
- 错误态/禁用态/聚焦态是否稳定可识别。

3. 信息层级
- 页面主次结构是否清楚。
- 关键操作按钮与次级操作是否抢占关系正确。

4. 可读性与密度
- 文本、标签、表格在 B 端场景下是否足够清晰。
- 是否存在冗余装饰导致噪音。

5. 交互一致性
- Dialog、Toast、确认交互是否一致。
- 同类操作在不同页面反馈是否一致。

## 阶段门槛

- 任一维度 < 3 分：该阶段不能收口，继续优化。
- 总分 < 20 分：不能进入下一阶段页面迁移。
- 必过项：表单体验、视觉一致性。
